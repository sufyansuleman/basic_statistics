4  Descriptive Statistics and Exploratory Data Analysis

4.1 When Do You Use This?

You have just received a dataset from a collaborator — 400 patients, 15 variables, and no codebook. Before running a single statistical test, you need to know what is in the data: which variables are continuous vs categorical, how many values are missing, whether any distributions are skewed, and whether any values look like data entry errors. That process is exploratory data analysis (EDA), and this session teaches you how to do it systematically.

4.2 Learning Objectives

After completing this session you will be able to:

  • Compute means, medians, standard deviations, IQRs, and missing value counts
  • Visualize distributions with histograms, boxplots, and density plots
  • Produce a publication-style Table 1 with gtsummary
  • Recognize skewed distributions and apply log transformations
  • Write a plain-English interpretation of a descriptive summary

4.3 Background: The EDA Mindset

Descriptive statistics summarize the data as it is. Inferential statistics make claims about a population. You must always do the first before the second.

The questions to answer in any EDA: 1. How many observations and variables? 2. What type is each variable (continuous, categorical, date)? 3. How many values are missing, and which variables? 4. What is the distribution of each continuous variable (symmetric, skewed, bimodal)? 5. Are there any values that seem implausible (outliers, data entry errors)?

4.4.5 Step 4: Examine a Key Variable — Bilirubin

# Raw distribution
ggplot(pbc, aes(x = bili)) +
  geom_histogram(bins = 30, fill = "steelblue", color = "white") +
  labs(title = "Serum bilirubin: raw scale",
       x = "Bilirubin (mg/dL)", y = "Count") +
  theme_bw()

The distribution is strongly right-skewed. This is common for lab values (bilirubin, ALT, creatinine). Log transformation often helps.

# Compare raw vs log-transformed
pbc_plot <- pbc |> filter(!is.na(bili))

p1 <- ggplot(pbc_plot, aes(x = bili)) +
  geom_histogram(bins = 30, fill = "steelblue", color = "white") +
  labs(title = "Raw scale", x = "Bilirubin (mg/dL)") +
  theme_bw()

p2 <- ggplot(pbc_plot, aes(x = log(bili))) +
  geom_histogram(bins = 30, fill = "darkorange", color = "white") +
  labs(title = "Log scale", x = "log(Bilirubin)") +
  theme_bw()

4.4.7 Step 6: Group Summary Table

pbc |>
  filter(!is.na(stage)) |>
  group_by(stage) |>
  summarise(
    n            = n(),
    pct_missing  = round(mean(is.na(bili)) * 100, 1),
    median_bili  = round(median(bili, na.rm = TRUE), 2),
    iqr_bili     = round(IQR(bili, na.rm = TRUE), 2)
  )

4.4.8 Interpreting the Output

Median bilirubin rises consistently from Stage 1 to Stage 4, from roughly 0.7 mg/dL to over 5 mg/dL. This pattern is consistent with progressive liver dysfunction: bilirubin is cleared by the liver, so rising serum levels reflect worsening hepatic function. The IQR also widens in later stages, indicating more heterogeneity among patients with advanced disease.

4.5 Worked Example 2: Plant Growth Study (Lab Data)

Not all research is clinical. This example demonstrates EDA on a simple designed experiment: three groups of plants treated differently.

data("PlantGrowth")

PlantGrowth |>
  group_by(group) |>
  summarise(
    n      = n(),
    mean   = round(mean(weight), 2),
    sd     = round(sd(weight), 2),
    median = round(median(weight), 2),
    iqr    = round(IQR(weight), 2),

4.7 What Can Go Wrong

Reporting means for skewed data Lab values (bilirubin, ALT, creatinine) are nearly always right-skewed. Report median and IQR, not mean and SD, unless you have confirmed approximate normality.

Missing data ignored in summaries mean(x) returns NA if any value is missing. Always use mean(x, na.rm = TRUE) — but also report how many values are missing.

Treating ordered categories as continuous Disease stage (1, 2, 3, 4) looks numeric but is an ordered factor. Treat it as a factor in plots and models unless you have a specific reason to assume linearity.

Table 1 with wrong test choice gtsummary defaults are sensible but check: does the p-value use a t-test when the variable is skewed? Use add_p(test = list(all_continuous() ~ "kruskal.test")) when distributions are non-normal.

4.9 Comprehension Check

  1. A variable has a mean of 45 and a median of 28. What does this tell you about the distribution?
  2. Why should you use mean(x, na.rm = TRUE) rather than mean(x) in most clinical datasets?
  3. You want to compare baseline characteristics between two treatment arms in a clinical trial. What type of table do you produce, and which R function creates it?
  4. A lab value has a right-skewed distribution. Which summary statistics are most appropriate to report?
  5. What does skim() from the skimr package show you that summary() does not?
Answers
  1. The mean is much larger than the median, which indicates a right-skewed distribution — a small number of very large values pull the mean upward. The median is the better measure of centre here.
  2. Without na.rm = TRUE, R returns NA for any vector that contains a missing value. Most clinical datasets have at least some missing values, so the result would be NA without this argument.
  3. A Table 1 (baseline characteristics table). In R, tbl_summary() from the gtsummary package creates this.
  4. Median and IQR. For right-skewed distributions, the mean and SD are misleading because they are pulled upward by the extreme values.
  5. skim() provides mini histograms for every numeric variable, making it easy to spot skewness at a glance. It also clearly separates numeric and character/factor variables and shows complete vs missing counts.
