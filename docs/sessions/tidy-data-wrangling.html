3  Importing, Cleaning, and Reshaping Data

Already know this?

If you can import data with read_csv(), filter and summarise with dplyr, and reshape between wide and long format with pivot_longer() / pivot_wider(), skip ahead to Part 2: Describing and Visualizing Data.

3.1 When Do You Use This?

You received a spreadsheet of mouse body weights measured every week for 12 weeks. Column names are Week1, Week2, …, Week12. Before you can run any analysis, you need to reshape this into a format R can work with — one row per measurement, not one row per mouse. This session teaches you the entire pipeline: import, inspect, clean, reshape, and prepare for analysis.

3.2 Learning Objectives

After completing this session you will be able to:

  • Import .csv and .xlsx files into R
  • Recognize the difference between “wide” and “tidy” (long) data formats
  • Use dplyr verbs to filter, select, and transform a data frame
  • Reshape data between wide and long format with pivot_longer() and pivot_wider()
  • Handle missing values using naniar
  • Join two data frames with left_join()

3.3 Background: What Is Tidy Data?

Most statistical functions in R expect data in tidy format: every row is one observation, every column is one variable. This is also called “long” format.

Data from lab instruments, spreadsheets, or electronic health records often arrives in “wide” format — especially repeated measurements over time, where each time point becomes a separate column.

--- Wide format ---
# A tibble: 2 × 4
  mouse_id Week1 Week2 Week3
  <chr>    <dbl> <dbl> <dbl>
1 M01       20.1  21.5  22.8
2 M02       21.3  22    23.1

--- Tidy (long) format ---
# A tibble: 6 × 3
  mouse_id week  body_weight
  <chr>    <chr>       <dbl>
1 M01      Week1        20.1
2 M01      Week2        21.5
3 M01      Week3        22.8
4 M02      Week1        21.3
5 M02      Week2        22  
6 M02      Week3        23.1
Note

The key rule of tidy data: one row = one observation. For a mouse measured weekly, each mouse-week combination is a separate observation.

3.4.2 Importing Data from Files

In real work you would import from a file rather than typing it in.

# CSV file
mouse_data <- read_csv(here::here("data", "mouse_weights.csv"))

# Excel file
library(readxl)
mouse_data <- read_excel(here::here("data", "mouse_weights.xlsx"), sheet = "Sheet1")
Tip

Always use here::here("data", "myfile.csv") instead of a full file path like "C:/Users/me/project/data/myfile.csv". The here package makes your code work on any computer regardless of where the project folder is located.

mouse_long <- mouse_wide %>%

3.4.5 Clean: Extract the Week Number

mouse_long <- mouse_long %>%
  mutate(week_num = as.numeric(str_remove(week, "Week")))

3.4.6 Filter, Select, and Mutate

# Filter: only high-fat mice from Week 8 onwards
hfd_late <- mouse_long %>%
  filter(group == "high_fat", week_num >= 8)

# Select: keep only the columns needed
analysis_cols <- mouse_long %>%
  select(mouse_id, group, week_num, body_weight)

# Mutate: set factor levels and add a derived column
mouse_long <- mouse_long %>%
  mutate(
    group     = factor(group, levels = c("control", "high_fat")),
    weight_kg = body_weight / 1000
  )

mouse_summary <- mouse_long %>%

penguins_clean <- penguins %>%

penguins_clean %>%

3.6 Joining Data Frames

It is common to have measurements in one file and metadata (group assignments, clinical variables) in another.

measurements <- tibble(
  mouse_id     = c("M01", "M02", "M03", "M04", "M05"),
  glucose_mmol = c(5.4, 5.1, 8.9, 9.3, 10.1)
)

metadata <- tibble(
  mouse_id = c("M01", "M02", "M03", "M04", "M05"),
  group    = c("control", "control", "hfd", "hfd", "hfd"),
  sex      = c("F", "M", "F", "M", "F")
)

combined <- left_join(measurements, metadata, by = "mouse_id")
combined
# A tibble: 5 × 4
  mouse_id glucose_mmol group   sex  
  <chr>           <dbl> <chr>   <chr>
1 M01               5.4 control F    
2 M02               5.1 control M    
3 M03               8.9 hfd     F    
4 M04               9.3 hfd     M    
5 M05              10.1 hfd     F
Join type Keeps rows from…
left_join(x, y) All rows from x, matched from y
inner_join(x, y) Only rows present in both
full_join(x, y) All rows from both
Tip

Use left_join() by default. Always check nrow() before and after to confirm no rows were unexpectedly dropped or duplicated.

3.7 Where to Find Data Like This

Mouse phenotype data: Mouse Phenome Database (phenome.jax.org) — hundreds of phenotyping studies with body weight, metabolic, and behavioral data. Data downloads as .csv.

Penguin data: palmerpenguins R package — install.packages("palmerpenguins"). Original data from Gorman et al. (2014).

Human clinical data: NHANES via the nhanesA R package:

library(nhanesA)
demo <- nhanes("DEMO_J")   # 2017-2018 demographics cycle

3.8 What Can Go Wrong

Column names with spaces or special characters Use janitor::clean_names() immediately after import: df <- read_csv("file.csv") %>% clean_names()

Silent type coercion If a numeric column has a stray "N/A" string, read_csv() imports it as character. Always check with glimpse() and fix with mutate(col = as.numeric(col)).

Row count changes after a join If the key is not unique in both tables, left_join() can duplicate rows. Run nrow(result) == nrow(original) after every join.

3.9.1 Exercise 1 (Guided): Reshape Clinical Trial Data

set.seed(100)
hr_wide <- tibble(
  patient_id = paste0("P", 1:8),
  treatment  = rep(c("drug", "placebo"), each = 4),
  Week0      = round(rnorm(8, 78, 8)),
  Week4      = round(rnorm(8, 74, 8)),
  Week12     = round(rnorm(8, 72, 8))
)
  1. Reshape to long format. New columns: "timepoint" and "heart_rate".
  2. Extract the numeric week into a new column week_num.
  3. Calculate mean heart rate per treatment x week_num.
  4. Plot mean heart rate over time for drug vs placebo.
Exercise 1: Solution 
hr_long <- hr_wide %>%
  pivot_longer(starts_with("Week"), names_to = "timepoint", values_to = "heart_rate") %>%
  mutate(week_num = as.numeric(str_remove(timepoint, "Week")))

hr_summary <- hr_long %>%
  group_by(treatment, week_num) %>%
  summarise(mean_hr = mean(heart_rate), se = sd(heart_rate)/sqrt(n()), .groups = "drop")

ggplot(hr_summary, aes(x = week_num, y = mean_hr, color = treatment, group = treatment)) +
  geom_line(linewidth = 1) +
  geom_point(size = 3) +
  geom_errorbar(aes(ymin = mean_hr - se, ymax = mean_hr + se), width = 0.4) +
  labs(title = "Heart Rate Over Time", x = "Week", y = "Mean heart rate (bpm)") +
  theme_bw()

3.10 Comprehension Check

  1. A data frame has 5 mice measured at 6 time points in wide format (one row per mouse). After pivot_longer(), how many rows will the result have?
  2. What does drop_na() do, and when should you use it with caution?
  3. Which join keeps all rows from the left table and adds matching columns from the right?
  4. After importing a CSV, a numeric column shows as <chr> in glimpse(). What most likely caused this?
  5. What is the purpose of group_by() followed by summarise()?
Answers
  1. 30 rows (5 x 6 = 30 observations).
  2. It removes any row containing at least one NA. Use it with caution in clinical data because patients with missing values may be systematically different from those without — dropping them can bias results.
  3. left_join(x, y, by = "id")
  4. A non-numeric character is present in that column (e.g., "N/A", ".", or a text note). read_csv() converts the whole column to character.
  5. group_by() splits the data into groups silently. summarise() computes a summary statistic within each group and returns one row per group.
